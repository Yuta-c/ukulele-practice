# ウクレレ練習アプリ

YouTube動画を再生しながら、現在のコード・次のコード・歌詞をリアルタイムで同期表示するウクレレ練習ツールです。スマートフォンでの使用を最優先にしています。

## セットアップ

```bash
cd ukulele-practice
npm install
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) を開きます。

## ビルドと検証

```bash
npm run build      # 本番ビルド
npm run lint       # ESLint（src/ を対象）
npm run type-check # TypeScript型チェック
npm start          # 本番サーバー起動
```

## 主な機能

### 動画読み込み
- YouTube URL（通常・短縮・共有・Shorts・Live）または動画IDを貼り付けて「読み込む」
- 再生・一時停止・5秒戻る・5秒進む・先頭へ戻る
- 再生速度: 0.5x / 0.75x / 1x / 1.25x（動画が対応している場合）

### 練習モード（▶ 練習ボタン）
- 現在のコードを非常に大きく表示
- 次のコードを右下に表示
- 現在の歌詞を中央に大きく表示、前後2行を薄く表示
- コード・歌詞をタップするとその時刻へジャンプ
- コードダイアグラム（表示/非表示切替可能）

### 編集モード（✏️ 編集ボタン）
- **コードタブ**: コードを現在の再生時刻に登録。時刻を±0.1秒調整可能
- **歌詞タブ**: 歌詞テキストを現在の再生時刻に登録。同様に調整可能
- **LRCタブ**: LRC形式のテキストを一括インポート/エクスポート

### LRC形式

```
[00:12.40][Am] 歌詞の1行目
[00:15.80][F] 歌詞の2行目
[00:20.00][C] 歌詞の3行目
```

コードのみ・歌詞のみの行にも対応します。

### 区間ループ（A-Bループ）
- 再生中に「現在時刻」ボタンでA点・B点を設定
- 「🔁 ループ」ボタンで有効化
- A点・B点は±0.1秒単位で微調整可能

### 曲データ管理
- LocalStorageに自動保存（ログイン不要）
- 「曲一覧」から複数の曲を管理（読み込み・複製・名前変更・削除）
- JSONファイルとしてエクスポート/インポート

### キーボードショートカット
| キー | 動作 |
|------|------|
| `Space` | 再生/一時停止 |
| `←` | 5秒戻る |
| `→` | 5秒進む |

※ 入力欄にフォーカス中はショートカットを無効化

### ウクレレコードダイアグラム
C, Cm, C7, D, Dm, D7, E, Em, E7, F, Fm, F7, G, Gm, G7, A, Am, A7, B, Bm, B7 に対応。
弦の並び: G-C-E-A（標準チューニング）

## 技術スタック

- **Next.js 14** (App Router)
- **TypeScript**
- **Tailwind CSS**
- **YouTube IFrame Player API**
- **LocalStorage**（外部DBなし・ログイン不要）

## 設計上の判断メモ

- **YouTube プレーヤーの永続化**: ビュー切替時にプレーヤー `<div>` を破棄すると再生が途切れるため、常にDOMに存在させ `compact` プロップで表示を切り替える方式を採用
- **コントロールオブジェクトの安定参照**: `useMemo` + 空の依存配列を使い、内部で `useRef` 越しにプレーヤーを操作することで、子コンポーネントや `useEffect` の依存配列に安全に渡せるようにした
- **コードと歌詞の分離管理**: LRC形式では同一タイムスタンプにコードと歌詞が共存することがあるが、内部データは独立した配列で管理し、インポート時にのみ結合を解消する
- **再生速度**: YouTube IFrame API は動画によって利用可能な速度が異なるため、`getAvailablePlaybackRates()` の結果をUIに反映する。デフォルトは [0.5, 0.75, 1, 1.25] を表示
- **自動再生制限**: ブラウザのAutoplay Policy対策として `cueVideoById`（自動再生しない）でロードし、ユーザー操作後に `playVideo` する方式を採用
- **コードダイアグラムのデータ**: GCEA標準チューニングの一般的なボイシングを採用。押さえにくいコード（E, B系）は高ポジションの一般的な形を使用
- **著作権**: アプリに既存楽曲の歌詞・コードを同梱しない。ユーザーが自身で入力・管理する設計
